Claudeとは？Anthropicの安全性重視LLMファミリー

読み：くろーど ／ 英：Claude

更新日：2026年6月10日 読了目安：約7分

Claude（クロード）は、Anthropicが開発する対話型大規模言語モデルファミリーです。OpenAIのChatGPT、GoogleのGeminiと並ぶ第三極として知られ、安全性と有用性のバランス——有害出力を抑えつつ実務で使える応答を届ける——を設計思想の中心に置きます。本記事は個別バージョンの性能比較より、開発元の区別と系列の読み方を試験向けに整理します。

試験で問われる見方

生成AIパスポートでは、ClaudeはChatGPT以外の主要な生成AIとしてシラバスで扱われます（TF-0142）。正解の骨格は「Anthropicが提供する主要な生成AIサービス・モデル群」です（HQ-0317、HQ-0316）。

Gemini・Copilotとの比較では、開発元の入れ替えに注意します。Claude＝Google、Gemini＝Anthropicのような選択肢は典型的な×パターンです。

演習で確認する

生成AIパスポート：TF-0142（Claudeの位置づけ）、HQ-0317（Claudeの役割説明）、HQ-0316（Gemini vs Claude）

Claudeとは

Anthropicは2021年に設立され、元OpenAI研究者らが中心となって安全なAIを掲げて立ち上げられました。Claudeはその対話型モデルの製品名・系列名で、Webアプリ、API、Slack連携、企業向け提供など複数の形で展開されます。

世代は Claude 1 → 2 → 2.1 → 3（Haiku / Sonnet / Opus）→ 3.5 → 3.7 → 4 と進み、試験では細部の暗記よりAnthropicのLLMファミリーとして答えられることが重要です。

安全性を前面にした設計

Claudeの差別化は「ベンチマーク1位」だけではなく、アライメント——人間の意図や価値観に沿った振る舞い——を重視する点にあります。AnthropicはRLHFに加え、Constitutional AI——原則に基づく自己修正——などを公表してきました。

観点試験での整理
安全性重視有害・違法な出力を拒否しやすい設計思想
有用性過度な拒否だけでは実務で使えない——両立が課題
安全性＝正確別問題。ハルシネーションは起こりうる
安全性＝アライメント関連するが同一ではない。アライメントは上位概念

G検定のAI倫理分野では、自動評価だけでなく安全性・事実性の確認が必要になる場面も問われます（G-350）。Claudeはその文脈で「安全志向の代表例の一つ」として位置づけられます。

詩の名前で示す階層

Claude 3以降、サイズ階層はHaiku（俳句）・Sonnet（十四行詩）・Opus（大作）という文学形式の名称で示されます。Googleの Ultra/Pro/Flash や OpenAIの GPT番号とは命名体系が異なる点が試験の整理ポイントです。

名称位置づけ試験での一言
Haiku最軽量高速・低コスト
Sonnet標準速度と性能のバランス
Opus最上位複雑な推論・分析

「3.5 Sonnet」「4 Opus」のように、世代番号＋階層名をセットで覚えます。Haikuだけではどの世代か特定できません。

モデル名と技術用語のすり替え

比較問題では、ClaudeをCNN・RNN・Transformerそのものと説明する選択肢に注意します。Claudeは生成AIの製品・モデル群であり、畳み込みや系列モデルといった基盤技術とは別レイヤーです（TF-0145は×）。

アーキテクチャの話（Transformer等）と、提供者の話（Anthropic）を混同しないことが、HQ-0316・HQ-0317系の問題で求められる力です。

よくある質問

ClaudeとChatGPTの違いは？

ClaudeはAnthropicのモデル・サービス群、ChatGPTはOpenAIの対話型サービスです。開発元と製品ラインを混同しないことが試験の核心です。

Haiku・Sonnet・Opusとは？

Claude 3以降のサイズ階層名です。Haikuは最軽量・高速、Sonnetは標準バランス、Opusは最上位性能——という役割で覚えます。試験では世代番号（3、3.5、4等）とセットで整理します。

Claudeは試験で出題されますか？

はい。生成AIパスポート第2章ではChatGPT以外の主要生成AIとして扱われ、「Anthropicが提供する主要な生成AIサービス・モデル群」として区別が問われます。
