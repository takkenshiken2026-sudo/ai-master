Claude 3 Sonnetとは？世代3の顔となる標準中核

読み：くろーど すりー そねっと ／ 英：Claude 3 Sonnet

更新日：2026年6月10日 読了目安：約7分

Claude 3 Sonnetは、Claude 3世代の標準モデルです。Claude 2.1が2系テキスト長文の完成形だったのに対し、3 Sonnetは世代更新——マルチモーダル対応と Haiku / Sonnet / Opus の三層——の中心に立つ顔でした。本記事はベンチマーク争いではなく、「なぜ最上位OpusではなくSonnetがデフォルトになるのか」に焦点を当てます。

試験で問われる見方

SonnetはClaude 3世代の標準帯として整理します。開発元はAnthropic（TF-0142）。Haiku / Sonnet / Opus は同一世代内のサイズ名であり、世代番号（3）と混同しないことが重要です。

Claude 3のマルチモーダルは「テキスト、画像、音声など複数種類の情報を扱える性質」（TF-0131、HQ-0315）の具体例です。マルチモーダルをRLHFと入れ替えた説明は×（HQ-0315）。

3 Sonnetとは

2024年3月、AnthropicはClaude 3を発表し、Sonnetを最初に広く提供する標準帯として位置づけました。公開時のベンチマークでは、前世代の最上位に相当する性能をSonnetが担う——つまり「標準帯が強くなった」——というナラティブが話題になりました。

Webアプリ（claude.ai）やAPIのデフォルト級モデルとして、最も多くのユーザーが触れたClaude 3の実体がSonnetであることが多いです。後続の 3.5 Sonnet が主力へ移行しても、「Sonnet＝標準帯」という命名規則は継承されています。

2系から3系への転換

観点Claude 2 / 2.1Claude 3 Sonnet
世代2系（2.1は改良版）3系（世代更新）
階層名単一系統が中心Haiku / Sonnet / Opus
入力主にテキストテキスト＋画像（マルチモーダル）
主な訴求100K〜200K長文性能刷新＋マルチモーダル

2.1の長コンテキスト路線は3系でも継承されますが、Sonnetの価値は「長さ」だけでなく総合能力の底上げ——特に画像を含む実務——にシフトした点にあります。

マルチモーダル標準帯

Claude 3 Sonnetは、チャートやスクリーンショット、手書きメモなどの画像入力を扱えるようになり、文書理解の幅が広がりました。試験では次を押さえます。

  • 定義 — 複数種類の情報を扱える性質（TF-0131）
  • 限界 — マルチモーダルでもハルシネーションは起こりうる
  • すり替え — マルチモーダル≠RLHF（HQ-0315）

Opusが最難推論向けでも、日常の画像付き業務——報告書の図表要約、UIスクショの説明——ではSonnetがコストと速度のバランスで選ばれます。

三層の使い分け

選ぶ目安向くモデル
大量の短い分類・タグ付けHaiku
日常の対話・文書要約・画像付き業務Sonnet
複雑な推論・研究レベルの分析Opus

「常にOpusが最適」ではありません。Gemini ProやGPT-4系列と同様、標準帯が実務のデフォルトになる設計が各社共通です。

性能とコストのスイートスポット

Sonnetが「世代3の顔」になる理由は、次のトレードオフにあります。

観点Haiku寄りSonnet寄りOpus寄り
応答品質単純タスクで十分多くの業務で十分難問で有利
速度最速バランス重くなりがち
APIコスト最低中程度高め
提供範囲バッチ向け広くデフォルト難問・限定用途

試験では数値の暗記より、Sonnet＝Claude 3の標準・バランスという役割対比が核心です。

よくある質問

Claude 3 Sonnetの位置づけは？

Claude 3世代の標準帯です。Haikuが最軽量、Opusが最上位の一方、Sonnetは多くのユーザーが日常で触れるバランス型として広く提供されました。試験では「Sonnet＝標準・バランス」と役割で区別します。

Claude 3 Sonnetと2.1の違いは？

世代が異なります。2.1は2系の改良版で主にテキスト長文、3 Sonnetは世代更新でマルチモーダル（画像等）と新しい三層（Haiku/Sonnet/Opus）を導入したモデルです。

Sonnetはマルチモーダルですか？

Claude 3世代は画像入力などマルチモーダル能力が強化され、Sonnetもその恩恵を受けます。ただしマルチモーダル＝誤りなしではなく、ハルシネーションや誤認識は起こり得ます。
