# 人気カードの書き方

**Web（公開中の記事リンク付き）:** https://ai-master.jp/docs/hub-featured/

一覧ページ上部の **「人気のページ」** カードの運用です（用語辞典・学習ガイド・キャリア共通）。

---

## やることは2つだけ

### 1. 載せる記事を決める

次の JSON に **記事IDを上から順に**書く。

| ページ | 編集ファイル |
|--------|--------------|
| 用語辞典 | `data/glossary-featured.json` |
| 学習ガイド | `data/guide-featured.json` |
| キャリア | `data/career-featured.json` |

例（用語辞典）:

```json
["claude-fable-5", "claude-mythos-5", "agi"]
```

### 2. ビルドを回す

```bash
python3 tools/build_glossary.py      # 用語辞典
python3 tools/build_guide_index.py  # 学習ガイド
python3 tools/rebuild_career.py    # キャリア
python3 tools/build_hub_featured_doc.py  # 運用ページ更新
```

---

## アイコンはどう決まる？

**基本は画像を置くだけ。** 設定ファイルに書くのは例外だけ。

| 順番 | 探す場所 | 例 |
|------|----------|-----|
| 1 | `data/hub-icon-aliases.json` に ID を書いた | 通常は空でOK |
| 2 | 記事フォルダ内の画像 | 下表「置き場所」 |
| 3 | カテゴリ用 SVG | 記事用が無いとき |

アイコンが無い記事も、**テキストだけのカード**で表示されます。

### 画像の置き場所（覚え方）

| ページ | 記事専用 | カテゴリ共通 |
|--------|----------|--------------|
| 用語辞典 | `assets/images/glossary/{ID}.svg` | `glossary/categories/{カテゴリ}.svg` |
| 学習ガイド | `assets/images/guide/{ID}/icon.svg` | `guide/categories/{カテゴリ}.svg` |
| キャリア | `assets/images/career/{ID}/role*.png` | `career/categories/{カテゴリ}.svg` |

**用語辞典だけの追加ルール:** Claude / GPT など開発元ロゴは `tools/` の既存 SVG を自動利用（`glossary` の `vendors` 参照）。

**学習ガイド:** `g-kentei-` で始まるカテゴリ → Gアイコン、`genai-` → Pアイコン（自動）。

**キャリア:** 47記事に `role_flow.png` などが既にあるので、多くは追加不要。
