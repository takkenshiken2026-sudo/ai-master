# 人気カードの書き方

**Web（公開中の記事リンク付き）:** https://ai-master.jp/docs/hub-featured/

一覧ページ上部の **「人気のページ」** カードの運用です（用語辞典・学習ガイド・キャリア共通）。

---

## やることは2つだけ

### 1. 載せる記事を決める

次の JSON に **記事IDを上から順に**書く。

| ページ | 編集ファイル |
|--------|--------------|
| 用語辞典 | `data/glossary-featured.json` |
| 学習ガイド | `data/guide-featured.json` |
| キャリア | `data/career-featured.json` |

例（用語辞典）:

```json
["claude-fable-5", "claude-mythos-5", "agi"]
```

### 2. ビルドを回す

```bash
python3 tools/build_glossary.py      # 用語辞典
python3 tools/build_guide_index.py  # 学習ガイド
python3 tools/rebuild_career.py    # キャリア
python3 tools/build_hub_featured_doc.py  # 運用ページ更新
```

## アイコンはどう決まる？

**基本は画像を置くだけ。** 設定ファイルに書くのは例外だけ。

| 順番 | 探す場所 | 例 |
|------|----------|-----|
| 1 | `data/hub-icon-aliases.json` に ID を書いた | 通常は空でOK |
| 2 | 記事フォルダ内の画像 | 下表「置き場所」 |
| 3 | カテゴリ用 SVG（`data/hub-category-icons.json` → `hub-icons/`） | 記事用が無いとき |

アイコンが無い記事も、**テキストだけのカード**で表示されます。

### 画像の置き場所（覚え方）

| ページ | 記事専用 | カテゴリ共通 |
|--------|----------|--------------|
| 用語辞典 | `assets/images/glossary/{ID}.svg` | `hub-icons/`（`hub-category-icons.json` で紐付け） |
| 学習ガイド | `assets/images/guide/{ID}/icon.svg` | 同上 |
| キャリア | `assets/images/career/{ID}/icon.svg` | 同上 |

**用語辞典だけの追加ルール:** Claude / GPT など開発元ロゴは `tools/` の既存 SVG を自動利用（`glossary` の `vendors` 参照）。

**学習ガイド:** `g-kentei-` で始まるカテゴリ → Gアイコン、`genai-` → Pアイコン（自動）。

**キャリア:** 人気カードは `icon.svg` またはカテゴリ SVG を使う（記事内の `role-flow.png` は図解用でカードには使わない）。

## カテゴリアイコン（Tabler ストローク）

共通デザインは `assets/images/hub-icons/`（**色付き背景 + 線画**）。一覧ではアイコン下にカテゴリ名ラベルを HTML で表示。再生成: `python3 tools/generate_hub_icons.py`。  
カテゴリとアイコンの対応は `data/hub-category-icons.json`。

### 用語辞典

`basics` · `models-tech` · `genai-use` · `data-ops` · `governance`

### 学習ガイド

`g-kentei-overview` · `genai-overview` · `compare` · `*-study` · `*-practice` · `*-domain` · `terms` · `audience` · `enterprise` · `trends` · `exam-tips` · `after-cert`

### キャリア

`role` · `learn` · `move` · `market`

## AIツールのアイコン（必須ルール）

**一覧には各ツールの公式アイコンのみ使う。** OpenAI / Google などメーカー共通 SVG は使わない。

| 優先 | 取得元 |
|------|--------|
| 1 | `assets/images/tools/{id}/app-icon.*`（記事ビルド時に公式URLから取得） |
| 2 | `assets/images/tools/{id}.{png,svg,...}` |
| 3 | `tools/build_tools_data.py` の `DIRECT_ICON_URL`（公式直リンク） |
| 4 | ツール公式ドメインの favicon / apple-touch-icon |

再取得:

```bash
python3 tools/build_tools_data.py --icons-only
```

Google / DuckDuckGo の favicon プロキシは**使用禁止**。
