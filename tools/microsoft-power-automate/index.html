Power Automateとは？DPA/RPAとMicrosoft 365連携自動化を解説

APIクラウドフロー（DPA）とUI操作RPA（デスクトップフロー）——M365・Dataverseに最適化されたiPaaS

DPA · RPA 1,000+コネクタ Copilot設計
対応環境
  • Web（Power Platform）
  • Windows（RPA）
  • Teams / SharePoint
更新日：2026年6月10日 読了目安：約10分

Microsoft Power Automate（旧 Microsoft Flow）は、Microsoft Power Platformの業務自動化（iPaaS＋RPA）サービスです。APIでSaaSをつなぐクラウドフロー（DPA）と、レガシーアプリをマウス・キーボードで操作するデスクトップフロー（RPA）を1製品で扱える点が最大の独自性です。本記事はコネクタ数の暗記ではなく、標準 vs Premiumコネクタのライセンス罠、Premium（ユーザー） vs Process（ボット）課金、Copilot in Power Automate ≠ M365 Copilot、Zapier/Makeとの棲み分け、試験の自動化ガバナンスに絞って整理します（2026年6月時点）。

試験で問われる見方

生成AIパスポートでは、Power Automateのような業務自動化は「効率化（○）」と「確認不要（×）」がセットで問われます。フローが外部SaaSに書き込みする場合、TF-0240（権限設計）、TF-0271（承認ステップ）、TF-0493（自動化後も確認）が実務と直結します。

試験用の整理：①DPA＝API連携、RPA＝UI操作——用語の混同に注意。②iPaaS（Zapier/Make）≠ AIエージェント（Manus）——Power Automateは前者に近いがCopilot/AI BuilderでAI要素あり。③M365に含まれる＝Premiumコネクタも使える（×）——ライセンス設計の典型陷阱。

Power Automateとは

Power Automateは、非エンジニアでも「トリガー→条件→アクション」のワークフローを組めるMicrosoftの自動化基盤です。1,000以上のコネクタ（2026年時点・Microsoft Learn）でOutlook、Teams、SharePoint、Salesforce、SQL Server等と連携し、Dataverse（Power Platform共通DB）上のデータも扱えます。

かつては「Microsoft Flow」という名称でした。現在はPower Platform（Power Apps / Power BI / Power Pages / Copilot Studio）と一体で、エンタープライズ向けにはManaged Environments（DLP、監査、ソリューション管理）が提供されます。

Zapierが「SaaS横断の連携数」で選ばれるのに対し、Power AutomateはMicrosoft 365テナント内の統治——Entra ID、コンプライアンス境界、Teams承認、Process Mining——で選ばれることが多いです。

DPA（クラウド）とRPA（デスクトップ）

Power Automate最大の試験・調達ポイントは、2種類の自動化エンジンを持つことです（Microsoft公式Pricing・Learn）。

方式 別名 仕組み 向くシステム
クラウドフロー DPA（Digital Process Automation） API・コネクタ経由。クラウド上で実行 M365、SaaS、Dataverse
デスクトップフロー RPA（Robotic Process Automation） Windows UI操作の記録・再生 APIのない社内アプリ、Citrix、レガシー

有人 vs 無人RPA

モード 必要ライセンス 実行イメージ
Attended（有人） Power Automate Premium（ユーザー） 担当者PC上でボットが支援しながら実行
Unattended（無人） Process または Hosted Process（ボット/台） サーバー/VM上で24時間自動実行

Hosted Process（$215/bot/月・年払い）はMicrosoft管理のAzure VM上で無人RPAを実行——インフラ運用を減らしたいEnterprise向けです（公式Pricing）。

フローの種類

Automated（自動）

イベントトリガー（メール受信、リスト更新等）で起動。所有者コンテキストで実行。

Instant（即時）

Teams/モバイルのボタンから手動起動。実行者コンテキストで実行——Premium要件に注意。

Scheduled（スケジュール）

日次・週次など定期実行。バッチ処理・レポート配信向け。

Business process（BPF）

Dataverse上の段階的業務プロセス（承認ステージ等）。CRM/案件管理と連動。

Desktop flow

Power Automate for desktopで作成するRPA。クラウドフローから呼び出し可能。

Process Mining

Premiumに含まれるプロセス可視化・ボトルネック分析（50MB/ユーザー、テナント最大100GB）。

標準 vs Premiumコネクタ

ライセンス設計で最も混乱するのがコネクタの種類です（Microsoft Learn FAQ）。

コネクタ 例 M365付属利用権 Premium単体
Standard Outlook、SharePoint、Teams、OneDrive ○（プランによる） ○
Premium Salesforce、SQL、HTTP、Oracle、SAP等 ×（別途Premium必要） ○
Custom 自社APIをOpenAPIで登録 × ○（Premium権限）

Automated/ScheduledフローでPremiumコネクタを使う場合、フロー所有者にPremiumライセンスが必要です。Instantフローでは実行する全ユーザーにPremiumが必要——人数が多い場合はフローにProcessライセンスを割り当てる方が経済的なこともあります（Microsoft Learn）。

Copilot・AI Builder

Power AutomateのAI要素は2系統です。混同しないことが試験・実務で重要です。

  • Copilot in Power Automate：デザイナー内で自然言語からフロー骨格を生成。「フォーム送信→Teams通知」のような指示からトリガー/アクションを提案（Microsoft公式）。
  • AI Builder：フォーム処理、オブジェクト検出、テキスト分類等のAIモデル。PremiumにAI Builder creditsが含まれ、フロー内アクションで消費。
  • Copilot Studio：別製品の対話型エージェント。エージェント内クラウドフローはCopilot Studioのメッセージ容量に含まれる場合あり（公式Community）——Power Automate Premium不要のケースあり。

Microsoft 365 Copilot（Word/Excel/TeamsのチャットAI）とは製品ラインが別です。試験では「Copilot＝すべて同じ」と一括りにしないでください。

よくある誤解

「Power Automate＝Microsoft 365 Copilot」は×です。前者はワークフロー自動化、後者は生産性アプリ内の対話AI——名称に「Copilot」が付く機能はPower Automate内にもありますが、別物です。

「M365 E3/E5に含まれるからPremiumコネクタも無料」は×です。多くのM365プランは標準コネクタ限定のPower Automate利用権です。Salesforce連携等にはPremiumライセンスが別途必要です。

「Power Automate＝ZapierのMicrosoft版だけ」も半分だけ正しいです。ZapierにないRPA・Process Mining・Dataverse BPFがPower Automateの差別化です。逆にZapierの9,000+連携・Zapier Agents・MCPはPower Automateより厚い領域もあります。

「自動化すれば人間確認不要」は×です。特にInstantフローで全社員がPremiumフローを実行できる設計は、TF-0493・TF-0271の「承認・確認」の反面教師になります。

「Processライセンス1つで同時に無限並列」——×です。1ボットは同時1実行。並列無人RPAには複数Process/Hosted Processが必要（公式Pricing注記）。

はじめ方・基本的な使い方

  1. make.powerautomate.com にサインイン Microsoft 365アカウントまたは30日トライアル。
  2. テンプレートまたはCopilotでフロー作成 「Recurrence→SharePoint取得→Teams投稿」等から開始。
  3. コネクタ種別を確認 Premiumアイコン付きコネクタはライセンス要件をチェック。
  4. テスト→ソリューションエクスポート Dev/Test/Prod環境はManaged Environments推奨。
  5. RPAが必要ならDesktop flow追加 クラウドフローから「デスクトップフローを実行」アクションで連携。
  6. 監視・Process Mining 失敗通知、Center of Excellence（CoE）キットでガバナンス。

よくある質問

Power Automateは無料で使えますか？

30日無料トライアルがあります。Microsoft 365の一部プランには標準コネクタ限定のPower Automate利用権が含まれますが、Premiumコネクタ・無人RPA・AI Builder本格利用にはPower Automate Premium（$15/ユーザー/月・年払い）またはProcessライセンスが必要です（2026年6月時点・公式Pricing）。

Power AutomateとMicrosoft 365 Copilotの違いは？

Microsoft 365 CopilotはWord/Excel/Teams等の対話型AIアシスタントです。Power Automateはトリガー→アクションのワークフロー自動化（iPaaS/RPA）プラットフォームです。Copilot in Power Automateはフロー設計を支援する機能の一つであり、製品全体を指しません。

クラウドフローとデスクトップフローの違いは？

クラウドフロー（DPA）はAPI/コネクタ経由のクラウド自動化。デスクトップフロー（RPA）はWindows上でキーボード・マウス操作を記録・再生するUI自動化です。Premiumは有人（attended）RPA、Processライセンスは無人（unattended）RPA向けです。

Power AutomateとZapierの違いは？

どちらもiPaaSですが、Power AutomateはTeams/SharePoint/Dataverse/Outlook等MicrosoftエコシステムとRPA（デスクトップフロー）・Process Miningが一体です。Zapierは9,000+ SaaS連携とZapier Agentsが強み。M365中心ならPower Automate、多SaaS API連携ならZapier——という棲み分けです。

Premiumコネクタには別ライセンスが必要ですか？

はい。Microsoft 365に含まれる利用権は標準コネクタ（Outlook、SharePoint等）が中心です。Salesforce、SQL、HTTP、カスタムコネクタ等のPremiumコネクタを使うクラウドフローは、所有者にPower Automate Premiumが必要か、フローにProcessライセンスを割り当てます（2026年6月時点・Microsoft Learn）。
