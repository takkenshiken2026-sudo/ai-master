Bolt.newとは？機能・料金・使い方をわかりやすく解説

ブラウザ上でフルスタックアプリをAIで丸ごと生成

無料枠あり AIコーディング ブラウザ完結
対応環境
  • Webブラウザ
更新日：2026年6月7日 読了目安：約8分

Bolt.new（ボルトニュー）は、StackBlitz（スタックブリッツ）が提供するブラウザ完結型のAIアプリビルダーです。自然言語のプロンプトからフルスタックアプリ（フロント・バックエンド・依存関係）を生成し、WebContainer技術でタブ内に開発環境を立ち上げてその場でプレビューできます。本記事は全機能の網羅ではなく、試験で押さえる位置づけ（AIコーディング支援・ブラウザ上の生成）とv0・Replit・Cursorとの違い、StackBlitz IDEとの混同に絞って書いています。料金や利用枠は2026年6月時点の情報です。利用前は料金プランと公式の最新情報を確認してください。

試験で問われる見方

生成AIパスポートやG検定では、AIコーディング支援や業務利用の注意点が問われることがあります。Bolt.newは「プロンプトからアプリを生成するブラウザツール」として、Cursor（ローカルエディタ）やChatGPT（汎用チャット）と並べて覚えると整理しやすいです。ただし試験では製品名そのものより、機密情報の入力禁止や利用規約・商用条件を問う設問の方が多い点に注意してください。

よくある誤解は3つあります。「Bolt.new＝StackBlitz IDE」と同一視すること（同社製品ですが、Bolt.newはAIによるアプリ生成に特化した別サービスです）。「Bolt.new＝v0と同じUI生成ツール」と思い込むこと（v0はUIコンポーネント中心、Bolt.newはフルスタック生成が強みです）。そして「AIが作ったアプリはそのまま本番に出してよい」と考えること（セキュリティレビューとテストが必須です）。

Bolt.newとは

Bolt.newは2024年頃から注目を集めたプロンプト駆動のAIアプリビルダーです。ユーザーが「ToDoアプリを作って」「Stripe連携の決済ページを追加して」と指示すると、AIがファイル構成・コード・依存関係を生成し、ブラウザ内の開発環境で即座に動作確認できます。ローカルへのNode.jsインストールやリポジトリのクローンが不要な点が大きな特徴です。

技術基盤はStackBlitzが開発したWebContainerです。WebAssembly上でNode.js互換のランタイムを動かし、npm installや開発サーバー起動をブラウザタブ内で完結させます。従来のオンラインIDEがクラウドVMに接続する方式と異なり、クライアント側で実行環境を構築するアプローチです。試験では「ブラウザ上で開発環境を動かすStackBlitzの技術」と覚えるとよいです。

提供元はStackBlitzです。同社はStackBlitz IDE（ブラウザ上のコードエディタ）でも知られますが、Bolt.newはAI生成に特化した別プロダクトです。既存プロジェクトをIDEで編集する用途と、ゼロからAIにアプリを作らせる用途は分けて理解してください。

できること（主な機能）

AIアプリ生成に関わる主な機能を整理します。

プロンプトからフルスタック生成

自然言語の指示からフロントエンド・API・設定ファイルまで一式を生成。React、Vue、Node.jsなどのスタックをAIが選定・構築します。

WebContainer上のライブプレビュー

生成直後にブラウザ内でアプリを起動。コード変更とプレビューが同一タブで完結し、環境構築の手間を省けます。

チャットによる反復編集

「ボタンの色を変えて」「認証機能を追加して」など、会話形式でアプリを改修。生成→確認→修正のループが短いです。

ファイルエクスポート・連携

生成したプロジェクトをダウンロードしたり、GitHub連携やデプロイ先へ公開したりできます（プラン・時期で機能は変動）。

依存関係の自動管理

package.jsonの更新やnpmパッケージのインストールをAIとWebContainerが自動処理。ライブラリ選定の負担を減らせます。

ノーコードに近い体験

コーディング経験が浅くてもプロトタイプを試せます。ただし本番運用にはコード理解とレビューが依然として必要です。

AIコーディングツールとの位置づけ

v0がUIコンポーネントの生成に強みを持つなら、Bolt.newはアプリ全体の生成と実行が中心です。ReplitはクラウドIDE＋ホスティングの総合プラットフォーム、CursorはローカルリポジトリのAI編集が本質、という整理が試験・選定の両方で役立ちます。いずれも生成AIのLLMを開発フローに組み込むツールですが、出力の粒度（部品かアプリ全体か）と実行環境（ブラウザかローカルか）で覚え分けましょう。

はじめ方・基本的な使い方

アカウント登録後、ブラウザだけで始められます。ローカル環境のセットアップは不要です。

  1. 公式サイトにアクセス bolt.new を開き、StackBlitzアカウントでサインインします（Google/GitHub連携など）。
  2. プロンプトでアプリを指示 例：「ReactとTailwindでシンプルなToDoアプリを作って」「天気APIを使うダッシュボードを追加して」
  3. 生成とプレビューを確認 AIがファイルを生成し、WebContainer上で開発サーバーが起動。プレビュー画面で動作を確認します。
  4. チャットで反復改修 デザイン変更・機能追加・バグ修正を会話形式で依頼し、都度プレビューで検証します。
  5. エクスポートまたはデプロイ 必要に応じてコードをダウンロード、GitHubへプッシュ、またはホスティング先へ公開します。

v0・Replit・Cursorとの比較

AIコーディング・ブラウザ開発で名前が並びやすい4つを整理します。

項目 Bolt.new v0 Replit Cursor
提供元 StackBlitz Vercel Replit Anysphere
製品の形 AIアプリビルダー（ブラウザ） UIコンポーネント生成 クラウドIDE＋ホスティング AIネイティブのローカルエディタ
主な強み フルスタックをプロンプトから生成 React UIの高速生成・プレビュー 多言語対応・実行・公開の総合環境 既存リポジトリの複数ファイル編集
実行環境 WebContainer（ブラウザ内） ブラウザプレビュー中心 クラウドVM ローカルPC
試験での覚え方 StackBlitz製・ブラウザ完結 Vercel製・UI生成 クラウドIDEの代表例 Anysphere製・ローカルエディタ

UI部品を素早く作るならv0、ゼロからアプリ全体をブラウザで試すならBolt.new、多言語のクラウド開発環境ならReplit、既存プロジェクトの日常開発ならCursor、という使い分けが現実的です。いずれも出力の検証は人が行う点は共通です。

よくある質問

Bolt.newは無料で使えますか？

はい。無料枠があり、プロンプトからアプリの生成・編集を試せます。ただしAI利用量やトークン、公開・デプロイ機能などに上限があり、継続利用では有料プランが必要になることが多いです。最新の枠と料金はBolt.new公式で確認してください。

Bolt.newとStackBlitzの関係は何ですか？

どちらもStackBlitz社の製品ですが、用途が異なります。StackBlitzは従来からブラウザ上でコードを編集・実行できるオンラインIDEです。Bolt.newは同社のWebContainer技術を土台に、AIがプロンプトからアプリ全体を生成・改修するサービスに特化しています。Bolt.new＝StackBlitz IDEだけ、という理解は誤りです。

Bolt.newとv0の違いは？

v0（Vercel）は主にReact UIコンポーネントの生成・プレビューに強みがあります。Bolt.newはフロントエンドに加えバックエンドやデータベース連携まで含めたフルスタックアプリを、ブラウザ内で生成・実行する点が中心です。UI部品を素早く作るか、アプリ全体を丸ごと作るか、という整理が試験・選定の両方で役立ちます。

Bolt.newとCursorの違いは？

CursorはローカルPCにインストールするAIネイティブのコードエディタで、既存リポジトリを開いて複数ファイルを編集します。Bolt.newはブラウザだけで完結し、プロンプトから新規アプリを生成する体験が中心です。既存プロジェクトの日常開発ならCursor、ゼロから素早く試作するならBolt.new、という使い分けが現実的です。

WebContainerとは何ですか？

StackBlitzが開発した、ブラウザ内でNode.jsなどの開発環境を動かす技術です。従来クラウドVMやローカルインストールが必要だった処理を、タブ内のWebAssembly上で実行できます。Bolt.newはこのWebContainer上で依存関係のインストールや開発サーバー起動まで行い、生成したアプリをその場でプレビューできる仕組みです。
