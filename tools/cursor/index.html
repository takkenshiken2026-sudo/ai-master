Cursor（カーソル）は、Anysphere（エニースフィア）が提供するAIネイティブのコードエディタです。Visual Studio Code（VS Code）をベースにしつつ、チャットから複数ファイルをまとめて編集できるComposerや、プロジェクト全体を参照するコードベースコンテキストが最初から組み込まれています。本記事は全機能の網羅ではなく、試験で押さえる位置づけ（AIコーディング支援・スタンドアロンエディタ）とGitHub Copilot・Windsurf・ChatGPTとの違いに絞って書いています。料金や利用枠は2026年6月時点の情報です。利用前は料金プランと公式の最新情報を確認してください。

試験で問われる見方

生成AIパスポートやG検定では、AIコーディング支援や業務利用の注意点が問われることがあります。Cursorは「チャットでコードを書き換えられるエディタ」として、ChatGPTでコード相談する例と並んで覚えると整理しやすいです。ただし試験では製品名そのものより、機密情報の入力禁止や利用規約・商用条件を問う設問の方が多い点に注意してください。

よくある誤解は3つあります。「Cursor＝ChatGPTのプラグイン」と思い込むこと（Anysphere製の独立したデスクトップエディタです）。「Cursor＝VS Code」と同一視すること（VS Codeフォークですが別アプリです）。そして「AIが書いたコードはそのまま本番に出してよい」と考えること（人のレビューとテストが必須です）。

Cursorとは

Cursorは2023年頃から注目を集めたAI前提で設計されたコードエディタです。従来のエディタにAI補完を足すのではなく、チャット・エージェント・コードベース参照が編集体験の中心に置かれています。開発者は自然言語で「このAPIのエラーハンドリングを追加して」と指示し、複数ファイルにまたがる変更案を受け取れます。

技術的にはVS Codeをフォークしたデスクトップアプリです。キーバインドや多くの拡張機能に慣れている人は移行しやすい一方、同期やライセンスはCursorアカウント側で管理されます。試験では「VS Codeの派生だが、AI機能が本体に統合された別製品」と整理すると混乱が減ります。

裏側のモデルはOpenAI、Anthropic、Googleなど複数のLLMを切り替えて利用できます（プランと時期で選択肢は変わります）。ChatGPTのWeb版でコード片を相談するのとは異なり、Cursorはローカルのプロジェクトフォルダを開いた状態でファイルを直接編集する点が本質的な違いです。

できること（主な機能）

コーディング支援に関わる主な機能を整理します。

AIチャット（Chat）

エディタ横のチャットでコードの説明・修正・リファクタリングを依頼できます。選択中のコードやエラー出力を文脈に含められます。

Composer（複数ファイル編集）

1つの指示から複数ファイルをまとめて変更するモード。機能追加やバグ修正をプロジェクト単位で進めやすいです。

コードベースコンテキスト

リポジトリ全体をインデックスし、関連ファイルをAIが参照しながら回答・編集します。

インライン編集（Cmd/Ctrl+K）

カーソル位置のコード块に対し、その場で生成・置換するショートカット編集が使えます。

タブ補完（Tab）

次に書きそうなコードを予測して提案する補完。GitHub Copilotに近い体験もCursor内に含まれます。

VS Code互換の拡張

多くのVS Code系拡張やテーマを利用できる場合があります（互換性は拡張ごとに要確認）。

AIコーディングツールとの位置づけ

GitHub Copilotが補完とチャット拡張を既存エディタに足すスタイルなら、CursorはエディタそのものがAIワークスペースです。Windsurfも同様にAIネイティブエディタとして比較対象になります。いずれも生成AIのLLMを開発フローに組み込むツールですが、製品の形（プラグインか専用エディタか）で覚え分けると試験・選定の両方で役立ちます。

はじめ方・基本的な使い方

デスクトップアプリをインストールし、プロジェクトフォルダを開くだけで始められます。

  1. 公式サイトからインストール cursor.com からOS向けインストーラを入手し、Cursorアカウントでサインインします。
  2. プロジェクトフォルダを開く 既存のGitリポジトリまたは新規フォルダを開き、コードベースのインデックスを待ちます。
  3. Chatで質問・修正を依頼 例：「この関数のユニットテストを追加して」「型エラーの原因を説明して」
  4. Composerで複数ファイル変更 機能追加などはComposerモードで関連ファイルをまとめて編集させます。
  5. 差分を確認してから適用 AI提案は必ず人がレビューし、テストを実行してからコミットします。

GitHub Copilot・Windsurfとの比較

AIコーディング支援で名前が並びやすい3つを整理します。

項目 Cursor GitHub Copilot Windsurf
提供元 Anysphere GitHub（Microsoft） Codeium（会社名は変更あり）
製品の形 AIネイティブの専用エディタ エディタ拡張・補完サービス AIネイティブの専用エディタ
主な強み Composer・コードベース参照 インライン補完の精度と普及 エージェント型の編集フロー
エディタ Cursor本体（VS Codeフォーク） VS Code・JetBrains等に追加 Windsurf本体（VS Code系）

今のエディタを変えず補完だけ欲しいならGitHub Copilot、プロジェクト単位でAIに編集させたいならCursorやWindsurf、コード片の相談や非エンジニア向けならChatGPT、という使い分けが現実的です。いずれも出力の検証は人が行う点は共通です。

よくある質問

Cursorは無料で使えますか？

はい。Hobby（無料）プランがあり、基本的なエディタ機能と限られたAIリクエストが利用できます。ただし高性能モデルやComposerの利用回数には制限があり、本格的に使う場合はProなどの有料プランが必要になることが多いです。最新の枠と料金はCursor公式で確認してください。

CursorとVisual Studio Codeの関係は何ですか？

CursorはVisual Studio Code（VS Code）をフォーク（派生）したデスクトップエディタです。見た目や拡張機能の互換性はVS Codeに近い一方、AIチャット・コードベース参照・複数ファイル編集などが最初から組み込まれた別製品です。VS CodeにAI拡張を足したものと同一ではありません。

CursorとGitHub Copilotの違いは？

GitHub Copilotは主にエディタ内のコード補完（オートコンプリート）やチャット拡張として動くAI支援です。CursorはエディタそのものがAI前提で設計され、チャットから複数ファイルをまとめて編集するComposerやコードベース全体の文脈参照が中心機能です。補完特化か、エディタ一体型か、という整理が試験・実務の両方で役立ちます。

CursorはChatGPTのプラグインですか？

いいえ。CursorはAnysphereが提供するスタンドアロンのコードエディタであり、ChatGPTのブラウザ版やプラグインではありません。裏側ではOpenAIやAnthropicなど複数のモデルを選んで使えますが、製品としては独立したデスクトップアプリです。ChatGPTでコード相談するのと、Cursorでプロジェクト全体を編集するのは用途が異なります。

Cursorは誰が提供していますか？

Anysphere（エニースフィア）が開発・提供しています。Microsoft製のGitHub CopilotやOpenAIのChatGPTとは別会社の製品です。試験では「AIコーディング支援ツールの一例」として名前が出ることがあり、提供元をAnysphereと覚えておくとよいです。
